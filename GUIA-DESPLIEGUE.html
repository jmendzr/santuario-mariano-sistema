🖥️ Opción A — Instalación Local (paso a paso)
🖥️ Instalar en Windows / Mac / Linux
1
Instalar Node.js 18+
Descargar de nodejs.org → instalar con opciones por defecto
2
Instalar PostgreSQL 14+
Descargar de postgresql.org/download → instalar. Anotar la contraseña del usuario postgres
3
Descomprimir el proyecto y configurar
Extraer el ZIP del sistema. Dentro de la carpeta backend/:
# Copiar y editar el archivo de configuración cp .env.example .env # Abrir .env y escribir su contraseña de postgres: DB_PASSWORD=su_contraseña_postgres RENIEC_TOKEN=su_token_de_apisperu.com
4
Instalar dependencias y crear la base de datos
cd backend npm install npm run setup-db # Crea tablas automáticamente
5
Iniciar el sistema
npm start # Abrir en el navegador: http://localhost:3000
6
💡 Para que arranque automáticamente con Windows
Instalar PM2 para que el servidor inicie con la PC:
npm install -g pm2 pm2-windows-startup pm2 start server.js --name "santuario" pm2 save pm2-startup install
🚀 Opción B — Railway en la nube (recomendado)
📋 Parte 1: Preparar el código en GitHub
1
Crear cuenta en GitHub
Ir a github.com → Sign up (gratis). Crear un nuevo repositorio llamado santuario-sistema
2
Subir el código del backend
Desde la carpeta backend/ del proyecto:
git init git add . git commit -m "Sistema Santuario v2" git remote add origin https://github.com/TU_USUARIO/santuario-sistema.git git push -u origin main
3
⚠️ Crear archivo .gitignore
Asegurarse de NO subir el archivo .env ni la carpeta uploads:
.env node_modules/ uploads/
🚂 Parte 2: Desplegar en Railway
1
Crear cuenta en Railway
Ir a railway.app → Login with GitHub. Plan Hobby: $5 USD/mes (≈ S/ 19). Incluye $5 de crédito de uso — suficiente para este sistema.
2
Crear nuevo proyecto
New Project → Deploy from GitHub repo → seleccionar santuario-sistema
3
Agregar PostgreSQL
En el panel del proyecto: + New → Database → PostgreSQL. Railway lo crea automáticamente y conecta las variables.
4
Configurar variables de entorno
En el servicio Node.js → Variables → agregar:
JWT_SECRET = clave_secreta_larga_aleatoria RENIEC_TOKEN = su_token_apisperu NODE_ENV = production # Las variables DB_* las agrega Railway automáticamente
5
Ejecutar el setup de base de datos
En Railway → su servicio → Settings → Deploy → Start Command:
node db/setup.js && node server.js
Esto crea las tablas la primera vez y luego inicia el servidor.
6
🎉 ¡Listo! Obtener la URL pública
Railway asigna una URL como: santuario-sistema.up.railway.app
Puede acceder desde cualquier celular o PC.

⚡
Quiero empezar ya mismo, sin complicaciones
Instalar todo local en Windows. En 20 minutos el sistema está corriendo. Perfecto para probar antes de ir a producción.

🏛️
El santuario tiene párroco, secretaria y varios colaboradores
Lo ideal es Railway: cada persona accede con su usuario desde su celular. No depende de que una PC esté encendida. El costo es menor al de una copia de llave mensual.
→ Opción Railway ⭐
🔌
Huarmey tiene cortes de internet frecuentes
La mejor solución es un servidor local (Local + PM2) con backups automáticos hacia la nube. El sistema funciona sin internet pero los datos se respaldan cuando hay conexión.
→ Combinación: Local + Backup en nube
🔍 Obtener Token RENIEC (APIsPerú) — Gratis
1
Registrarse en APIsPerú
Ir a apisperu.com → Registrarse con email (es gratis). Plan gratuito incluye 100 consultas de DNI por mes — suficiente para un santuario.
2
Copiar el Bearer Token
En el panel de APIsPerú → sección "Token" → copiar el token generado.
3
Pegarlo en el archivo .env
RENIEC_TOKEN=eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiJ9... (su token aquí) RENIEC_API_URL=https://api.apisperu.com/api/dni
